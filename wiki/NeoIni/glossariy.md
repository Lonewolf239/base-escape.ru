# 📖 Глоссарий · NeoIni

| Термин | Определение |
|--------|-------------|
| **NeoIniDocument** | Главный класс, предоставляющий API для работы с INI‑конфигурацией. Потокобезопасен, поддерживает горячую перезагрузку, шифрование и проверку целостности. |
| **NeoIniOptions** | Класс параметров, передаваемый в конструктор. Управляет авто‑добавлением, интервалом авто‑сохранения, использованием контрольной суммы, экранированием и др. |
| **INeoIniProvider** | Интерфейс для подключаемых хранилищ (файл, база данных, удалённый ресурс). Провайдер отвечает за ввод‑вывод, а документ — за парсинг, шифрование и логику. |
| **NeoIniFileProvider** | Стандартная реализация `INeoIniProvider`, работающая с файловой системой. Поддерживает резервные копии, контрольные суммы и шифрование. |
| **IEncryptionProvider** | Интерфейс для кастомных алгоритмов шифрования. Позволяет заменить встроенный AES на собственную реализацию. |
| **IHotReloadMonitor** | Интерфейс для мониторинга изменений источника данных. По умолчанию используется опрос контрольной суммы провайдера. |
| **EncryptionType** | Перечисление, задающее режим шифрования: `None` – без шифрования, `Auto` – автоматический ключ (привязан к машине), `Custom` – пароль или кастомный провайдер. |
| **HumanMode** | Режим ручного редактирования. Сохраняет комментарии, пустые строки и форматирование. Отключает контрольные суммы и шифрование, так как они мешают ручному изменению. |
| **Shielding** | Экранирование значений двойными кавычками при записи (например, `key = "value ; not comment"`). Позволяет использовать символы `;` и `=` внутри значений. |
| **Checksum** | Контрольная сумма содержимого, сохраняемая в конце файла. Позволяет обнаружить непреднамеренные изменения и автоматически восстановиться из резервной копии. |
| **Hot Reload** | Автоматическая перезагрузка конфигурации при изменении источника. Активируется вызовом `StartHotReload()`. |
| **AutoSave** | Автоматическое сохранение после каждой операции записи (`SetValue`, `AddSection` и т.д.). Управляется свойствами `UseAutoSave` и `AutoSaveInterval`. |
| **AutoAdd** | Автоматическое создание отсутствующих секций и ключей при чтении через `GetValue<T>` (если включено). Для чтения без побочных эффектов используйте `TryGetValue<T>`. |
| **Clamped Methods** | Методы, ограничивающие значение заданным диапазоном: `GetValueClamped<T>` и `SetValueClamped<T>`. Выходящие за границы значения приводятся к ближайшей допустимой границе. |
| **AsyncReaderWriterLock** | Внутренняя блокировка, обеспечивающая потокобезопасность: несколько читателей одновременно, писатели — исключительно. Поддерживает асинхронные операции. |
| **Source Generator** | Генератор исходного кода на этапе компиляции (пакет `NeoIni.Generators`). Создаёт строго типизированные методы `Get<T>()` и `Set<T>()` для моделей, помеченных атрибутом `[NeoIniKey]`. |
| **NeoIniKeyAttribute** | Атрибут (пакет `NeoIni.Annotations`), который связывает свойство модели с конкретной секцией, ключом и значением по умолчанию. |
| **NeoIniData** | Публичный класс, представляющий внутреннюю структуру данных (`Dictionary<string, Dictionary<string, string>>`). Используется при реализации кастомных провайдеров. |
| **Black Box Philosophy** | Принцип проектирования: пользователь взаимодействует только с публичным API `NeoIniDocument`, никогда не касаясь внутренних структур. Упрощает код и изолирует приложение от деталей хранения. |
| **PBKDF2** | Алгоритм формирования ключа шифрования из пароля с использованием соли. Применяется в стандартной реализации AES. |
| **Salt** | Случайные 16 байт, генерируемые для каждого файла при шифровании. Обеспечивает уникальность ключа даже при одинаковых паролях. |
| **CancellationToken** | Поддержка отмены во всех `Async` методах (загрузка, сохранение, горячая перезагрузка). |
| **Provider** | Общее название компонента, реализующего `INeoIniProvider`. Позволяет NeoIni работать с файлами, базами данных или любым другим хранилищем. |
| **UseAutoBackup** | Опция файлового провайдера, создающая резервную копию с расширением `.backup` перед каждой записью. При ошибке чтения данные автоматически восстанавливаются из резервной копии. |
